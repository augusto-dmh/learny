# Learny v2 Retrospective (RFC-002, Cycles A–G)

**Date**: 2026-07-17
**Scope**: The seven cycles of [RFC-002](../rfc/0002-learny-v2-roadmap.md) that took Learny from a
deterministic, network-free MVP (v0.1.0) to a deployed product with real AI providers, PDF ingestion,
active recall, a real frontend, and an automated GHCR → VPS deploy (v0.2.0).

Each cycle ran as a `tlc-spec-driven` slice — specify → design → tasks → execute — with an independent
Verifier re-deriving coverage from the spec (author ≠ verifier), atomic per-task commits, and a small
reviewed PR. Decisions are recorded as ADRs; per-cycle decision logs live in `.specs/`.

## What shipped, cycle by cycle

### Cycle A — `v2-foundation` (PR #17, tagged `v0.1.0`)

The first v2 cycle laid the release and CI foundation rather than a product feature: it landed the
public-facing docs (README, QA runbook, research evidence, RFC-002), a 4-job GitHub Actions CI
workflow, Apache-2.0 licensing and OSS hygiene (SECURITY, CONTRIBUTING, dependabot), and four
correctness fixes surfaced by the pre-cycle end-to-end QA (F1 compose worker-storage, F2 storage-fault
classification, F3 proxy header hygiene, F4 first-run migration determinism). Verifier PASS (19/19 ACs,
6/6 sensor mutants, one fix iteration); review triage found 12 issues, 11 real and all fixed — including
two phantom CI action versions and a MinIO-in-CI gap the live run confirmed. Final CI green across 507
tests against real Postgres + Redis + MinIO services.

### Cycle B — `v2-embeddings` (PR #20)

Replaced the hashed deterministic embeddings with real semantic retrieval: an `OpenAIEmbeddingAdapter`
(`text-embedding-3-large` at `dimensions=1536`, fitting the existing `vector(1536)` column) behind the
existing `EmbeddingPort`, per-chunk model versioning, an idempotent/resumable re-embed task, and a
tier-2 recall@k / MRR evaluation ([ADR-0019](../adr/0019-use-openai-embeddings-with-per-chunk-model-versioning.md)).
It also fixed F8 with language-aware full-text search (a per-row `search_config` trigger, proven with a
Portuguese stemming case). Verifier PASS (22/22 ACs, 5/5 mutants); review found 4 real issues, all fixed.
The deterministic adapter stayed the CI/offline default — the first paid SDK entered the tree without
making the suite require a key.

### Cycle C — `v2-generation` (PR #23)

Brought real prose: Anthropic Claude adapters for both answers and teaching behind `AnswerGenerationPort`
and `TeachingGenerationPort`, with native citations (one citations-enabled document per evidence chunk),
a whole-reply `NOT_FOUND_IN_SOURCE` sentinel that fixed the F5 relevance-aware not-found gap, SSE
streaming via a Learny-owned domain event stream rendered as UI Message Stream v1 at the edge, and an
LLM-judge evaluation harness (faithfulness + relevancy) wired into a nightly, secret-gated `eval.yml`
([ADR-0020](../adr/0020-use-anthropic-claude-for-generation.md)). Verifier PASS (23/23 ACs, 6/6 mutants).
This was the fifth consecutive backend-only slice — deliberate, because the streaming consumer was
scheduled as Cycle D.

### Cycle D — `v2-frontend` (PR #24)

Rebuilt the UI layer on Tailwind v4 + shadcn/ui + vendored AI Elements, wired real streaming through the
AI SDK `useChat` against the Cycle-C UI Message Stream endpoints via the same-origin proxy, added the
in-book reader (`/sources/[id]/read?anchor=`) and a single new read endpoint, and restored the app shell
(sidebar library with status badges, theme toggle, ingestion polling). Verifier PASS (22/22 ACs, 6/6
sensor mutants across both stacks). This closed the five-cycle backend-only streak and restored the full
vertical-slice cadence.

### Cycle E — `v2-active-recall` (PR #25)

The flagship feature, shipped as one full slice: a 4-table quiz schema (content / scheduling / history /
jobs) with no corpus FK (citation snapshots survive re-ingest), `QuizGenerationPort` with deterministic
and Anthropic Batch-API adapters plus server-side quality checks, a `SchedulingPort` over FSRS-6, a
Celery deck pipeline with re-ingest reconciliation, review endpoints, an Anki-shaped review UI, and
genanki `.apkg` export ([ADR-0021](../adr/0021-active-recall-design.md)). Verifier PASS (25/25 ACs, 6/6
mutants); gates green at 833 backend / 174 frontend tests.

### Cycle F — `v2-ingestion-breadth` (PR #26)

Closed the format gap and a structure-quality debt in one integrated change: a Docling PDF parser behind
the renamed `DocumentParserPort`, a pure format-agnostic `normalize_book` pass (title inference, flat-TOC
hierarchy re-derivation, depth clamping, trivial-section merge with anchor aliasing, Gutenberg-marker
stripping — the F7 fix), composite deterministic PDF anchors with page-span citations, and an isolated,
memory-bounded `worker-pdf` service with models baked into its image
([ADR-0022](../adr/0022-pdf-ingestion-via-docling-and-corpus-normalization.md)). Verifier PASS (24/24 ACs,
7/7 mutants); backend 941 passed / 11 skipped, with CI proven to stay green docling-free.

### Cycle G — `v2-deploy` (this cycle)

Delivered the runtime and the presentation finale: a `deploy.yml` gated on green CI via `workflow_run`
that publishes three GHCR images (`learny-backend`, `learny-pdf-worker`, `learny-web`) and scp+ssh-deploys
them to a VPS, prod compose consuming those images via `${LEARNY_IMAGE_TAG:-latest}`, a Caddy TLS edge as
the single public surface (only 80/443, certs persisted, proxying solely to `web:3000`), nightly eval
JSONL committed to a dedicated `eval-results` branch, a VPS deploy runbook, the version bump to 0.2.0, and
this retrospective ([ADR-0023](../adr/0023-ghcr-ssh-deploy-caddy-edge.md)). This resolves the TLS /
reverse-proxy portion of TDD open question #10; backup and monitoring remain deferred by prior decision.

## What worked

- **Spec-driven cycles with an independent Verifier.** Author ≠ verifier caught coverage gaps the
  implementer's own mental model would have missed; every cycle reports a clean ACs-matched + sensor-mutant
  result, and the discrimination sensor (inject a fault, confirm a test kills it) kept the suites honest
  rather than merely green.
- **Deterministic-first adapters.** Keeping network-free adapters the default behind every port meant CI
  and local development stayed offline and key-free through the entire migration to paid providers — the
  suite never required an OpenAI or Anthropic key to pass, and provider outages can't redden a PR.
- **Ports and adapters made provider swaps mechanical.** Each real provider (OpenAI, Anthropic ×3 ports,
  Docling, FSRS) landed as a new adapter class selected by a `LEARNY_*_PROVIDER` setting and wired in one
  composition root — no domain or application code changed to accommodate a vendor.
- **Small, reviewed PRs with atomic commits.** One PR per cycle, phase-scoped sub-agent workers, and a
  self-contained git history kept reviews tractable and rollbacks precise; review triage consistently
  converted findings into fix commits rather than deferred debt.

## What to improve

- **The long backend-only streak.** Cycles A–C were backend/config/docs slices, deliberately deferring the
  UI consumer to Cycle D — five consecutive slices without a user-visible surface (each flagged at its merge
  gate). It was defensible sequencing, but it delayed end-to-end feedback on streaming and real generation
  until late; interleaving a thin UI touch earlier would have surfaced integration friction sooner.
- **Real-provider evaluation snapshots are still deferred.** The committed eval snapshots are deterministic
  (`local-deterministic@1536`); the real `text-embedding-3-large` and Claude baselines, and the judge
  threshold gate, wait on a keyed run. The nightly `eval.yml` now persists JSONL to `eval-results`, so the
  next step is to calibrate thresholds from accumulated real-provider history and turn the gate on.
- **Demo media is still a manual author task.** The capture guide and named asset slots ship in-cycle, but
  the GIF and screenshots need real books, live keys, and a human-paced browser session — the README carries
  a placeholder until they're recorded.
- **Prod image hygiene follow-ups.** The `runtime` backend image still installs the `dev` extra, and the
  `worker-pdf` image is multi-gigabyte with slow rebuilds after GHA cache eviction — both accepted this
  cycle and recorded in ADR-0023 as follow-ups.

## Forward look

Post-v2 candidates, all currently out of RFC-002 scope and none yet decided (recorded as possible
directions, not commitments):

- Multi-provider / bring-your-own-key generation and embeddings.
- A dedicated vector database or a reranker, if PostgreSQL hybrid search stops scaling.
- Notes and second-brain workflows layered on the canonical corpus.
- A monitoring/metrics stack and automated backups (structured logs and the ops runbooks are today's hooks;
  the backup/monitoring part of TDD open question #10 stays open).
- Scanned-PDF OCR (Docling `do_ocr` is off for born-digital focus) and localized normalization heuristics.
- An `eval-results` dashboard rendering the nightly JSONL history.
- A publicly hosted, multi-tenant instance.
